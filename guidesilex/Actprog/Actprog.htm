Guide de l’utilisateur Win-Silex  Activités diverses

Activités programmables

L’accès au module se fait par la barre de menu : commande Configuration act. Div. Dans le menu Act. Diverses




Gestion prévisionnelle des activités

Vous arrivez alors dans l’écran d’accueil du module Gestion prévisionnelle activités. De là, vous pouvez créer soit des Activités libres, soit des Activités programmables, en cliquant sur le bouton correspondant. Par ailleurs, chacune des parties de l’écran contient la liste des activités déjà crées. Nous nous intéressons ici uniquement aux activités programmables.



 Configuration d'une activité

Après avoir cliqué sur le bouton Ajout de la colonne Activités programmables, vous accédez à l’écran d’accueil du sous-module Description générale de l'activité.

Dans un premier temps, vous devez nommer l’activité en utilisant la zone de saisie Nom activité. Vous pouvez également définir un Code si vous souhaitez relier plusieurs activités programmables entre elles.




Pour accéder à la construction de l'activité vous devez cliquez sur le bouton Détail activité.

Cette action fait apparaître deux onglets supplémentaires en haut de l'écran que nous allons décrire dans les paragraphes qui suivent.


















Description des tableaux





Une activité est composée de plusieurs tableaux eux-mêmes composés de plusieurs lignes.

Le nombre maximum de tableaux par activité est de 6.

Le nombre maximum de lignes par tableau est de 19.


Pour chaque tableau, on doit renseigner:

  • le nom du tableau ex: lapins, volailles,...

  • type du tableau: Annuel ou Autres

Il y a deux types de tableaux: - annuel = les colonnes sont représentées par les années d'étude

- autres = les colonnes sont à définir

  • Nb lignes de 0 à 19

  • Nb Cols de 0 à 14 si tableau de type Autres sinon le nombre de colonnes est défini par le nombre d'années.

  • Page imp: définit le numéro de page et la position du tableau dans la page.

Le chiffre des centaines correspond au numéro de page.

Le chiffre des unités correspond à la position du tableau dans la page.

ex: 101 => page 1 position 1

201 => page 2 position 1

202 => page 2 position 2



Description des lignes


Pour les tableaux annuels:





Intitulé

Intitulé de la ligne.


Config.

définit le type de la ligne

  • 1. Saisie: ligne saisie.

  • 2. Calculée: ligne calculée à partir d'autres lignes.

  • 3. Espace: ligne d'espacement fine purement esthétique.

  • 4. Titre: ligne d'espacement large avec titre purement esthétique.

  • 5. Ligne ref.: ligne faisant référence à une ligne d'un autre module.

  • 6. Accès deb.: ligne calculée mais la première année de l'étude est saisie.

  • 7 Accès fin.: ligne calculée mais la dernière année de l'étude est saisie.

  • 8 Bilan: ligne calculée dont le résultat est envoyée dans une ligne du bilan.


Nb Dec

définit le nombre de décimales de la ligne de 0 à 4.


Formule

Formule de la ligne si la ligne est calculée.

Les utilisateurs de tableurs ne seront pas perturbés car les formules sont du même type.

Le numéro se situe à droite de chaque ligne.



La syntaxe utilisée pour l'élaboration des formules est la suivante:


C1

Rappel de la colonne 3 ligne 1 (idem tableur)


#[Num. de ligne]

Rappel de la ligne en question

Ex: #1 + #2 = fait la somme de la ligne 1 et 2


@[Num. de ligne]+[Nb col]

Rappel de la ligne en question décalée vers la droite du nombre de colonnes

Ex: @2+1= rappelle la ligne 2 décalée d'une année


@[Num. de ligne]-[Nb col]

Rappel de la ligne en question décalée vers la gauche du nombre de colonnes


@[Num. de ligne]:[Nb col]

Rappel de la ligne et de la colonne en question

Ex: @2:3 = rappelle la ligne 2 année 1

Attention, les colonnes 1 et 2 sont réservées à l'intitulé et au logiciel.


If(A,B,C)

Si A est vrai alors B sinon C

Opérateur: >,<,=

Ex: If(#1>#2,#1,#2)


ROUND (B,D)

arrondi B au chiffre le plus près en utilisant le nombre de décimales définit par D.

Ex: ROUND(#1/#2,2)


F11

l'utilisation de la touche F11 permet de choisir une ligne de l'étude pour la rapatrier dans

une ligne définie comme Ligne réf.

Il est également possible d'importer les données d'une ligne d'une autre activité

programmable en utilisant une formule du type [Act(C2)-100] dans laquelle C2 est le code

de l'activité dans laquelle on va rechercher la donnée et 100 est le numéro de la ligne

concernée.


Police

Type de la police: - Normal

                  • Gras

                  • Italique

                  • Gras Italique

                  • Invisible (permet de cacher une ligne)


Couleur

Permet de choisir la couleur de la ligne.


=> Vers cpt. Res

En appuyant sur F11 lorsqu'on se trouve sur cette case la liste des lignes impactables au compte de résultat apparaît, il suffit alors d'en choisir une.

Si la ligne est une ligne de bilan, la touche F11 permet de choisir une ligne parmi les lignes impactables du bilan.



Style imp.:

Définit le style d'impression de la ligne: - Normal

                  • Avec cadre

                  • Non imprimé



Pour les tableaux autres:




Il faut faire la même chose pour les colonnes et les lignes.



















Utilisation et construction de l'activité après paramétrage des tableaux:


Une fois le paramétrage effectué, il faut construire l'activité. Pour cela, il faut cliquer sur le bouton Enregistrer situé en bas de l'écran et vous vous retrouvez en mode utilisation de l'activité.





Vous pouvez revenir au paramétrage des tableaux en cliquant sur le bouton Configuration activité en bas de l'écran.

Si vous souhaitez conserver les données saisies en mode utilisation, vous devez utiliser le bouton Enregistrer avec données après chaque modification des paramétrages de tableaux au lieu d'Enregistrer seulement.
