Marge animaux libre

    L'accès au module se fait par la barre de menu : commande Marge ani. libre Dans le menu Act. Animales, cliquez sur Ajout.

Gestion prévisionnelle Nouvelle activité

Le module Marge animaux libre est conçu sur le modèle d'une activité allaitante dans laquelle tout est paramétrable.

Vous avez ainsi la possibilité de construire des activités avec ou sans reproducteurs. Ainsi, le module permet de gérer des activités telles que :

Un troupeau

  • Elevage d'équins, de cervidés, de bisons, d'autruches, etc...
  • Elevage de bovins ou d'ovins qu'il est utile de différencier dans une étude (conduite de troupeau différente, race différente, etc...)

Un atelier d'engraissement sans reproducteurs

  • Engraissement de taurillons achetés ;
  • Gavage de canards ;
  • Etc...

Vous avez la possibilité de créer autant d'activités animaux libres que vous le souhaitez dans la limite de 9.

Chacune de ces activité ne peut toutefois pas être reliée au module de consolidation fourragère.

Dans l'écran d'accueil Gestion prévisionnelle Nouvelle activité, vous devez commencer par saisir dans les cases blanches les informations qui permettent d'identifier l'activité dans votre étude. Les boutons qui suivent, sous le titre Caractéristiques du troupeau, permettent d'accéder aux différents tableaux du module. C'est au niveau de chacun de ces tableaux que vous pouvez paramétrer l'ensemble de l'activité. Les boutons ayant un titre vert reproduisent les tableaux qui existent dans le module vaches allaitantes. Les boutons qui portent un titre rouge sont spécifiques à ce module. Ils vous permettent de définir l'ensemble des animaux qui existent au sein de l'atelier. Vous devez définir non seulement les caractéristiques des reproducteurs, mais aussi celles des produits dont vous envisagez la commercialisation ainsi que tous les types d'animaux qui peuvent exister sans forcément être commercialisés (il s'agit par exemple des animaux élevés pour le renouvellement).

Par ailleurs, un ensemble de cases à cocher vous permet de préciser l'existence :

  • De reproducteurs ;
  • De productions annexes ;
  • De femelles ;
  • De mâles.

    Une fois dans un des tableaux, des onglets vous permettent de passer rapidement de l'un à l'autre, mais vous pouvez également utiliser le bouton Tab. Suivant en bas à droite de l'écran. Le bouton Fin vous permet de revenir à l'écran Gestion Prévisionnelle Nouvelle activité quel que soit le tableau dans lequel vous travaillez.

Définition des reproducteurs :

En cliquant sur le premier bouton au texte rouge intitulé Définition des animaux, vous accédez à l'écran Définition des reproducteurs.

Nom reproducteurs

Les case blanches vous permettent de personnaliser les Noms des reproducteurs mâles et femelles. Des libellés sont saisis par défaut. En outre, vous devez définir la Destination au compte de résultat de chaque catégorie d'animaux. Le bouton marqué d'un double chevron en bout de ligne vous donne accès à une fenêtre dans laquelle vous pouvez choisir l'une des adresses possibles au compte de résultat. Par défaut, toutes les ventes issues de l'activité seront affectées au compte de résultat sous le titre Produit div 1.

Attention

La définition d'une adresse au compte de résultat est obligatoire. Dans le cas contraire, les renseignements afférents au type d'animaux que vous avez créé ne seront pas pris en compte dans l'étude.

Paramètres des animaux de renouvellement

Ce titre permet de définir l'âge à la mise bas. Le choix se fait par un menu déroulant accessible par un bouton situé en bout de ligne. Trois choix sont possibles :

  • Moins d'un an ;
  • Un à deux ans ;
  • Plus de deux ans.

    Types d'animaux

    Les cases blanches disponibles à ce niveau permettent de personnaliser les noms des différents types d'animaux élevés pour le renouvellement en fonction de leur âge. La liste qui vous est proposée dépend du choix retenu pour l'âge à la mise-bas.

    Comme pour le nom des reproducteurs, vous devez obligatoirement définir une Destination au compte de résultat.

Définition des produits :

En cliquant sur le bouton intitulé définition des produits, vous accédez à une fenêtre vous permettant de gérer la crétion de nouveaux produits ou leur suppression. Les produits en question sont liés à l'existence de l'atelier. Il peut s'agir, par exemple de prestations de services ou de sous produits (type laine ou autre ...). Il ne s'agit pas d'animaux à commercialiser.

Le bouton Ajout permet de créer une ligne correspondant à un nouveau produit. Vous devez ensuite renseigner un certain nombre de critères qui caractérisent le produit que vous venez de créer :

  • Le nom du Produit ;
  • L'Unité ;
  • Une case à cocher permet de préciser si le produit fait l'objet d'une Cession vers un atelier de transformation ;
    Attention
    Pour l'instant, la connexion entre modules n'est pas opérationnelle pour les cessions. Ainsi, si vous prévoyez une cession depuis un atelier créé avec le module animaux libres, vous devez veiller à reporter manuellement cette cession dans l'atelier de destination.
  • La Destination au compte de résultat, sur le même principe que décrit plus haut.

Définitions des animaux produits :

En cliquant sur le deuxième bouton au texte rouge intitulé Définition des animaux, vous accédez à l'écran Définition des catégories d'animaux produits. Cet écran vous permet de définir les catégories d'animaux qui sont commercialisés par l'atelier. Une distinction est faite entre les mâles et les femelles (le passage de l'un à l'autre se fait en cliquant sur l'onglet correspondant). La définition d'un type d'animal pour un sexe donné se fait de façon unique par le biais de plusieurs critères. Chacun correspond à un tableau particulier accessible par une barre de menu.

Durée de vie

  • Catégorie Mâles (ou Catégorie Femelles), il s'agit du nom par lequel vous identifiez le type d'animal venu dans l'étude ;
  • Age Ent., est l'âge auquel l'animal apparaît ; dans la plupart des cas, il va naître sur l'exploitation et cette case sera vide ou contiendra la valeur zéro ; par contre, si un chiffre apparaît, l'animal n'est pas né sur l'exploitation, il est donc acheté à un âge correspondant à la valeur que vous avez indiquée ; il peut aussi être cédé par un autre atelier.

Attention


    Pour l'instant, la connexion entre modules n'est pas opérationnelle pour les cessions. Ainsi, si vous prévoyez une cession vers un atelier créé avec le module animaux libres, vous devez veiller à reporter manuellement cette cession dans l'atelier d'origine.
    • Mois Ent., il s'agit du mois où l'animal fait son apparition sur l'atelier (mois de naissance ou mois d'achat) ;
    • Dur. Vie est la durée de vie du type d'animal concerné, depuis sa naissance jusqu'à sa vente. Cette valeur est égale à la durée de présence de l'animal lorsque celui-ci est né sur l'exploitation. Elle est supérieure lorsque l'animal a été acheté à l'extérieur et qu'il a été fini sur l'exploitation.

    Consommation :

    La saisie se fait en Kg et correspond à la totalité des aliments consommés sur la vie de l'animal (c'est-à-dire sur le nombre de mois où celui-ci est présent sur l'exploitation). Vous pouvez choisir entre trois types d'aliment :

    • Céréales, elles peuvent être consommées quelque soit le type d'animal ;
    • Alt. -1 an, ce type d'aliment ne concerne que les animaux de moins d'un an, dans le cas d'animaux vendus avant leur premier anniversaire la saisie est impossible dans la case Alt. Autre ; dans les autres cas, la saisie ne concerne que la première année de vie de l'animal ;
    • Alt. Autre, ce type d'aliment ne concerne que les animaux de plus d'un an, dans le cas d'animaux nés sur l'exploitation et vendus à plus d'un an l'Alt. Autre n'est consommé qu'après leur premier anniversaire.

    Vous n'êtes toutefois pas obligé de saisir les consommations à ce niveau. En effet, vous pouvez effectuer une saisie globale par catégorie d'animal au niveau de l'onglet Consommations.

    Attention

    Les valeurs des consommations déjà inscrites dans le tableau de l'onglet Consommations. sont prioritaires (même si elles sont nulles). Si vous avez saisi des consommations spécifiques à chaque type d'animaux créé et que vous voulez mettre à jour ce tableau (ou simplement le remplir), il est nécessaire de lancer l'opération manuellement au moyen du bouton Calcul des consommations situé dans l'onglet Consommations. Sans cette opération, Win-Silex peut travailler avec des valeurs incohérentes sans vous donner de message d'avertissement.

    Le calendrier de vie de l'animal

    En double-cliquant sur la case qui porte le nom du type d'animal, vous faites apparaître un tableau qui figure le scénario affecté au type d'animal correspondant en terme de durée de vie et d'alimentation. L'exemple ci-dessous simule un atelier d'embouche. Les premiers veaux destinés à devenir des Boeufs peuvent être achetés en Décembre 2004 (achat à 6 mois), ils atteignent l'âge de 12 mois en Mai 2005 (ce qui est mis en évidence par un changement de couleur), l'âge de 24 mois en Mai 2006 et seront vendus en Décembre 2006, soit à la fin du onzième mois du troisième exercice de l'étude.

    La partie basse de l'écran correspond aux consommations. Les 1 800 Kg de céréales définis comme paramètres sont répartis sur la durée de vie de l'animal, au prorata de son temps de présence sur l'exploitation. L'aliment moins de un an est intégralement affecté à la première année de vie de l'animal (soit ici sur ses six premiers mois de présence sur l'exploitation). L'aliment autre est réparti sur la durée de vie de l'animal, au prorata de son temps de présence sur l'exploitation après son premier anniversaire. Cette répartition est mensuelle, il n'est pas possible d'autres phases d'alimentation que "moins de un an" et "plus de un an".

    ATTENTION

    La définition d'un scénario est relativement rigide puisque vous fixez la date et l'âge d'entrée de chaque type d'animal ainsi que sa durée de vie, donc par déduction sa date de sortie (un type d'animal donné ne peut être vendu qu'à une date bien précise). Vous pourrez donc être amené à définir plusieurs types différents pour des animaux très proches dans la réalité.

    Dans l'exemple présenté ici, les Boeufs ne peuvent être issus que d'animaux achetés et en aucun cas d'animaux nés sur l'exploitation (achat au mois de Septembre d'animaux âgés de 6 mois). Dans le cas où vous envisageriez la vente de boeufs de 30 mois nés sur l'exploitation, vous devriez obligatoirement créer un deuxième type de boeuf dont l'âge d'entrée serait zéro.

    Cession

    Ce tableau permet de définir si l'animal fait l'objet d'une cession interne ou externe, par exemple vers un atelier de transformation. La cession peut être à l'entrée (ex : cession de génisses grasses à un atelier d'embouche) ; elle peut aussi être à la sortie (ex : cession de génisses grasses à un atelier de transformation). Le choix se fait par le biais de cases à cocher.

    Attention


      Pour l'instant, la connexion entre modules n'est pas opérationnelle pour les cessions. Ainsi, si vous prévoyez une cession depuis un atelier créé avec le module animaux libres, vous devez veiller à reporter manuellement cette cession dans l'atelier de destination.

    Destination au compte de résultat

    La définition d'une adresse au compte de résultat est obligatoire. Dans le cas contraire, les renseignements afférents au type d'animaux que vous avez créé ne seront pas pris en compte dans l'étude.

    La création d'un nouveau type d'animal se fait en cliquant sur le bouton Ajout.

    Le bouton Sup. Permet la suppression d'un type d'animal existant.

    Le bouton Tri permet de classer les types d'animaux par durées de vie croissantes.

Paramètres

Caractéristiques

Les coefficients UGB sont à renseigner ou à modifier, ils servent pour la détermination des effectifs zootechniques ainsi que pour l'expression de la marge.

Vous pouvez également opter pour un coefficient différent de l'UGB. Dans ce cas vous devez renseigner la ligne correspondant au titre Coeff. Charges diverses. Il sert au calcul des charges opérationnelles et éventuellement comme unité pour l'expression de la marge.

Paramètres divers

Expression de la marge

Dans l'ensemble du module, les marges pourront être exprimées : soit en fonction des UGB, soit en fonction du Coeff. Charges diverses que vous avez pu définir plus haut. Vous pouvez sélectionner une des deux options en cliquant sur le bouton correspondant. Attention, ce choix doit être cohérent avec les renseignements indiqués au titre Caractéristiques animaux.

Consommations MS ou UF / UGB

Win-Silex permet de réaliser des calculs de bilan fourrager. Le paramètre que vous indiquerez ici correspond à la consommation pour une UGB et pour une année. Vous pouvez faire le choix entre deux unités de calcul : l'UF ou le Kg de Matière Sèche. Il est bien entendu inutile de renseigner ce critère si vous ne vous référez pas à la notion d'UGB ou si les animaux élevés ne consomment pas de fourrages.

ATTENTION

La consommation MS ou UF /UGB peut vous être demandée pour différents types d'animaux. Vous devez retenir l'option pour l'UF ou le KG de Matière Sèche une fois pour toutes. Il est important de veiller à la cohérence de vos choix dans les différents modules et sous-modules pour que le bilan fourrager global de l'exploitation soit juste.

Reproducteurs

L'onglet Reproducteurs se décompose en deux tableaux : Effectifs repro. et Production.

Vous accédez à chacun d'entre eux en cilquant sur le bouton correspondant.

Effectif reproducteurs

Ce tableau permet la gestion des effectifs du troupeau reproducteur.

Plusieurs boutons permettent d'accéder à des niveaux de détail supplémentaires.

Valeur inventaire

Un clic sur le bouton en haut à gauche portant l'intitulé permet d'accéder au tableau Valeur inventaire. Deux zones sont à renseigner : Effectifs début et Valeurs / animal. Les valeurs pour chaque catégorie d'animal seront retenues pour l'élaboration des bilans financiers dans l'ensemble de l'étude et les effectifs sont ceux du bilan d'ouverture.

Reproducteurs

Cette partie du tableau permet la gestion des effectifs de femelles année par année.

Sur la première ligne, vous devez saisir l'effectif de femelles présentes au début de chaque année de l'étude, la première valeur étant la même que celle figurant dans le tableau Valeur Inventaire (le remplissage se fait simultanément).

Les Sorties annuelles de femelles sont saisies sur la deuxième ligne. Un clic sur le bouton à droite du titre permet d'accéder à un niveau de détail supplémentaire. Vous pouvez ainsi ventiler le chiffre des sorties de l'année entre les animaux réformés et les pertes (pour un même nombre de sorties, les pertes viennent diminuer le montant des ventes).

Les valeurs qui apparaissent dans la ligne Entrées correspondent à l'intégration annuelle des animaux de renouvellement élevés sur l'exploitation (qu'ils soient nés sur la ferme ou élevés à partir de femelles achetées jeunes).

La ligne Achats est une valeur calculée. Elle correspond au nombre de femelles de renouvellement qu'il est nécessaire d'acheter chaque année pour pouvoir atteindre l'Effectif début de l'année suivante une fois pris en compte le nombre des Entrées. La ligne Femelles eff. Fin est un simple rappel.

ATTENTION

L'Effectif fin de la dernière année de l'étude n'est pas une valeur calculée, vous devez donc saisir une valeur. Dans le cas contraire, le calcul des achats conduit à une valeur incohérente (achats négatifs).

Les lignes consacrées aux mâles fonctionnent sur le même principe que celles consacrées aux femelles, y compris pour la saisie de l'Effectif fin de la dernière année d'étude

Renouvellement

Par défaut, les animaux élevés pour le renouvellement mettent bas à un âge défini au titre des paramètres des animaux de renouvellement. Néanmoins vous pouvez définir des mises-bas anticipées : pour cela il suffit d'en saisir le nombre dans la case correspondante. Les animaux sont pris sur l'effectif disponible pour l'année concernée, ils viennent s'ajouter au nombre des mises-bas à un âge "normal".

Vous pouvez enfin acheter ou vendre des animaux de renouvellement. Les valeurs qui apparaissent dans le tableau sont des effectifs globaux par année, l'accès au détail des achats et des ventes par tranche d'âge se fait en cliquant sur le bouton situé à droite du libellé (voir figure ... ).

Effectifs

La ligne Femelles eff. Moyen renvoie des valeurs calculées.

L'effectif moyen de femelles est calculé de la façon suivante : (effectif début + effectif fin) / 2. Cette valeur entre dans le calcul du nombre d'UGB ou d'animaux de l'atelier (calcul de chargement, bilans fourragers, etc...). Si vous le souhaitez, vous pouvez entrer une valeur différente de l'effectif calculé automatiquement. Pour cela, il vous faut cliquer sur le bouton Calcul à droite du libellé ce qui a pour effet de réinitialiser la ligne et de rendre la saisie possible.

Production

l'Effectif moyen productif est une valeur calculée. Il est obtenu en ajoutant à l'effectif moyen calculé, le nombre des entrées de l'année.

La ligne Nb de naissances / an renvoie des valeurs calculées à partir de l'Effectif moyen productif et du coefficient de Productivité (Effectif moyen productif X Productivité). Si vous voulez, vous pouvez entrer une valeur différente de celle calculée automatiquement pour le nombre de naissances. Pour cela, il vous faut cliquer sur le bouton Calcul à droite du libellé, ce qui a pour effet de réinitialiser la ligne et de rendre la saisie possible.

Par défaut, tous les animaux nés sont des Mâles. La répartition mâles / femelles se fait par la saisie de la dernière ligne du tableau.

Effectif femelles

Ce tableau permet la gestion des effectifs de femelles destinées à la vente (à l'exception des réformes qui sont gérées dans le tableau Effectif reproducteurs). Plusieurs boutons permettent d'accéder à des niveaux de détail supplémentaires. Ceux-ci sont destinés à faciliter le contrôle de cohérence nécessaire dans la construction de chaque année. Des allers-retours entre le tableau des Effectifs femelles et ces divers sous-tableaux peuvent s'avérer nécessaires pour aboutir à des mouvements d'animaux cohérents.

La ligne Cession femelles rappelle le nombre de femelles nées chaque année tel que vous l'avez défini dans le tableau Eff. Repro.

La ligne Dispo. à la vente renvoie pour chaque année le nombre de femelles qui n'ont pas été affecté à une utilisation particulière (vente, renouvellement). Un clic sur le bouton à droite du titre permet d'accéder à un tableau récapitulatif des ventes et des cessions internes destinées au renouvellement. Ce niveau de détail supplémentaire est destiné à vous aider à trouver une destination possible pour les femelles Disponibles.

Valeur inventaire

Un clic sur le bouton en haut à gauche permet d'accéder au tableau Valeur inventaire. Les zones Effectif début et Valeur animal permettent de saisir les effectifs présents en début d'étude par tranche d'âge ainsi que les valeurs à retenir pour l'élaboration des bilans financiers dans l'ensemble de l'étude.

Les zones suivantes (Eff. Renouv., Eff. vendu, Eff. acheté, Achat renouv., Eff. calculé) permettent un contrôle de cohérence du devenir de l'effectif d'animaux présent en début d'étude. Le résultat des différents mouvements d'animaux (achats, ventes et cessions internes destinées au renouvellement) doit conduire à un effectif calculé rigoureusement identique à l'Effectif début. Dans le cas contraire, des valeurs négatives incohérentes peuvent apparaître dans le tableau Effectif femelles.

Stocks

Cette partie du tableau permet en particulier de réaliser un contrôle de cohérence sur l'ensemble des ventes de femelles. Ainsi, si des erreurs existent, elles conduiront à faire apparaître des effectifs négatifs. D'autre part, dans l'exemple présenté ici, il n'est prévu aucune vente d'animaux âgés de plus de trois ans ; par conséquent, la ligne Stocks +3 ans doit toujours être vide (si vous oubliez des ventes, Win-Silex sera conduit à accumuler des stocks d'animaux de plus de trois ans qui n'auront aucune possibilité de sortir de l'atelier puisqu'il n'a pas été défini de produit correspondant).

Ventes

Cette partie du tableau permet de saisir les effectifs d'animaux vendus chaque année. Les nombres proposés doivent être en cohérence avec le tableau des Stocks et des Valeurs inventaires.

ATTENTION

Vous ne pouvez vendre que des animaux correspondant à des catégories que vous avez définies au début du module. Ainsi, dans l'exemple présenté ici, si vous souhaitiez vendre des animaux de plus de trois ans, il serait nécessaire de commencer par créer la catégorie correspondante en revenant au début du module.

L'utilisation de l'ascenseur au bas du tableau est fortement conseillée : elle permet de visualiser le devenir des animaux présents en fin d'étude et qui n'ont pas encore été vendus (Cf. Utilisation des boutons +3 ans et -3 ans dans le module vaches allaitantes). Vous pouvez ainsi terminer le contrôle de cohérence sur les ventes et les stocks. Dans le cas où ce contrôle ne serait pas effectué, des valeurs aberrantes peuvent persister.

Achats à destination

En fonction des ventes prévues au cours de l'étude, des achats sont provoqués automatiquement en fonction de la définition de la catégorie d'animal.

Cette définition stricte peut amener à définir deux catégories d'animaux différentes qui correspondront au même type de produit à la vente, mais qui auront deux origines différentes.

Bilan effectifs

Aucune saisie n'est possible dans ce tableau. Il s'agit d'un simple rappel de la partie Stocks des trois tableaux précédents (Effectifs reproducteurs, Effectifs femelles et Effectifs mâles).

La ligne Total animaux indique l'effectif total, toutes catégories confondues pour chaque année de l'étude.

Enfin, un calcul en UGB est proposé en bas du tableau. Il n'est renseigné que si vous avez complété le tableau des correspondances des UGB dans l'onglet paramétrage.

Consommations

Cet onglet récapitule les quantités consommées annuellement par chacun des Reproducteurs (mâles et femelles) d'une part, et par l'ensemble des Femelles et l'ensemble des Mâles destinés à la vente d'autre part. Ces quantités sont exprimées en Kg pour les différents types d'aliments : Céréales, Aliments -1 an, Aliments autres. Il se décompose en plusieurs tableaux accessibles par des boutons spécifiques situés dans un bandeau en haut de l'écran.

En ce qui concerne les animaux reproducteurs, vous devez saisir les consommations annuelles par femelle et par mâle. En ce qui concerne les Femelles et les Mâles destinés à la vente, vous pouvez transférer automatiquement les consommations paramétrées au début du module (Cf. § Gestion Prévisionnelle Nouvelle Activité), en utilisant le bouton Calcul des consommations. Toutefois, vous pouvez aussi saisir directement les consommations annuelles par type d'animal dans le tableau (par exemple, dans le cas où vous n'auriez pas défini de consommation pour chaque type d'animal crée au début du module).

Un Bilan en bas du tableau vous rappelle les besoins annuels de l'atelier.

Attention

Les valeurs des consommations déjà inscrites dans le tableau de l'onglet Bilan Conso. sont prioritaires (même si elles sont nulles). Si vous avez saisi des consommations spécifiques à chaque type d'animaux créé et que vous voulez mettre à jour ce tableau (ou simplement le remplir), il est nécessaire de lancer l'opération manuellement au moyen du bouton Calcul des consommations. Sans cette opération, Win-Silex peut travailler avec des valeurs incohérentes sans vous donner de message d'avertissement.

Marge

Produits ; Charges

La marge se compose de deux tableaux principaux : Produits et Charges auxquels on accède en cliquant sur les boutons portant le libellé correspondant. Ces deux tableaux présentent une synthèse des résultats conduisant à la constitution de la marge, ils obéissent au code de couleurs général de Win-SILEX (voir présentation générale). Par défaut, le module s'ouvre sous le tableau Produits.

Pour chaque type d'animal par tranche d'âge, un bouton à droite du libellé permet de faire apparaître un sous-tableau dans lequel le produit est détaillé suivant les achats, les ventes par catégories d'animaux définies et les variations d'inventaire.

La ligne Primes renvoie par défaut un résultat calculé à partir des informations que vous avez inscrites dans le tableau Paramètres et dans le tableau Primes accessible à partir de la Marge. La ligne Autres primes permet de saisir des subventions qui ne seraient liées ni aux UGB, ni aux droits à produire.

Primes

Le bouton correspondant ouvre un tableau qui permet la saisie de l'ensemble des primes relatives à l'atelier.

Prix repro. ; Prix fem. ; Prix mâles

Ces boutons, accessibles à partir du tableau Produits, permettent la saisie des prix correspondant aux différentes catégories d'animaux, à l'achat ainsi qu'à la vente.

Prix divers

Ce tableau, disponible à partir du tableau Charges, permet la saisie des prix unitaires correspondant aux différentes charges affectées à l'atelier : frais vétérinaires, frais d'élevage, aliments, etc...
